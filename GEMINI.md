# Modern Design Patterns Agent (GEMINI.md)

## Purpose
This MCP server implements advanced 2025–2026 design patterns, focusing on **AI-Agentic Orchestration**, **Cellular Resilience**, and **Sustainability (GreenOps)**. It provides a standardized way for agents and systems to operate according to modern architectural constraints.

## Usage for Agents
- **`sequential_chain`**: Use this to define a sequence of processing steps, where Agent A's output is Agent B's input.
- **`ensemble_voting`**: Use this to gather multiple perspectives (from different agents or models) and run a "quorum" or "reviewer" logic to find the best output.
- **`shared_memory`**: Use this as a persistent context layer to store and retrieve state across multiple disparate agents.
- **`cellular_resilience`**: Use this to manage and define "Cells" of a system to control the "blast radius" of failures.
- **`carbon_aware_scheduler`**: Use this to check and simulate task scheduling based on the "carbon intensity" of the local grid.

## Design Philosophy
- **Agentic**: Treats agents as autonomous micro-services.
- **Cellular**: Prioritizes isolation and shared-nothing architectures.
- **Sustainable**: Carbon intensity is a first-class scheduling constraint.
